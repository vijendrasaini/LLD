# Standard LLD Problem Solving Approach

For every problem:

## Step 1 - Gather Requirements

* [ ] Ask features
* [ ] Ask assumptions
* [ ] Ask constraints

---

## Step 2 - Identify Use Cases

* [ ] User actions identified
* [ ] System actions identified

---

## Step 3 - Identify Core Objects

* [ ] Extract nouns
* [ ] Identify entities

Example:

Parking Lot

* Vehicle
* Slot
* Ticket
* Gate

---

## Step 4 - Assign Responsibilities

* [ ] Data ownership identified
* [ ] Action ownership identified
* [ ] Communication ownership identified

---

## Step 5 - Draw UML

* [ ] Association
* [ ] Composition
* [ ] Aggregation
* [ ] Inheritance

---

## Step 6 - Apply Design Patterns

* [ ] Pattern justified
* [ ] Pattern improves design

---

## Step 7 - Code

* [ ] Core entities implemented
* [ ] Clean code
* [ ] SRP followed

---

## Step 8 - Discuss Improvements

* [ ] Scalability
* [ ] Extensibility
* [ ] Concurrency
* [ ] New Features
